Detailed Description

A trading range financial plot.

See also
See Trading Range Plot for a list of animatable properties.
See Trading Range Plot Bindings for a list of supported binding identifiers.

Method Documentation

- (nonnull instancetype) initWithFrame: (CGRect)  newFrame
implementation

Initializes a newly allocated CPTTradingRangePlot object with the provided frame rectangle.

This is the designated initializer. The initialized layer will have the following properties:

  • plotStyle = CPTTradingRangePlotStyleOHLC
  • lineStyle = default line style
  • increaseLineStyle = nil
  • decreaseLineStyle = nil
  • increaseFill = solid white fill
  • decreaseFill = solid black fill
  • barWidth = 5.0
  • stickLength = 3.0
  • barCornerRadius = 0.0
  • showBarBorder = YES
  • labelField = CPTTradingRangePlotFieldClose

newFrameThe frame rectangle.

The initialized CPTTradingRangePlot object.

- (BOOL) pointingDeviceDownEvent: (nonnull CPTNativeEvent *)  event
atPoint: (CGPoint)  interactionPoint 
implementation

Informs the receiver that the user has pressed the mouse button.

If this plot has a delegate that responds to the -tradingRangePlot:barTouchDownAtRecordIndex: or -tradingRangePlot:barTouchDownAtRecordIndex:withEvent: methods, the interactionPoint is compared with each bar in index order. The delegate method will be called and this method returns YES for the first index where the interactionPoint is inside a bar. This method returns NO if the interactionPoint is outside all of the bars.

- (BOOL) pointingDeviceUpEvent: (nonnull CPTNativeEvent *)  event
atPoint: (CGPoint)  interactionPoint 
implementation

Informs the receiver that the user has released the mouse button.

If this plot has a delegate that responds to the -tradingRangePlot:barTouchUpAtRecordIndex: and/or -tradingRangePlot:barTouchUpAtRecordIndex:withEvent: methods, the interactionPoint is compared with each bar in index order. The delegate method will be called and this method returns YES for the first index where the interactionPoint is inside a bar. This method returns NO if the interactionPoint is outside all of the bars.

If the bar being released is the same as the one that was pressed (see -pointingDeviceDownEvent:atPoint: ), if the delegate responds to the -tradingRangePlot:barWasSelectedAtRecordIndex: and/or -tradingRangePlot:barWasSelectedAtRecordIndex:withEvent: methods, these will be called.
